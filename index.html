Platform Guide

What is Horizon QA?

Horizon QA is an automated quality assurance platform for the Horizon AI Concierge. It tests how well the Concierge handles different types of visitor queries by sending prompts, capturing responses, and scoring them against expected behavior using an LLM-as-judge pattern.

Test Suite Types

Q&A Prompts test the Concierge's ability to answer factual questions about the destination -- things like restaurant recommendations, parking info, attraction hours, and activity suggestions. Each prompt gets sent to the Concierge, the response is captured, and a judge evaluates accuracy, relevance, and helpfulness. You can optionally provide an "expected response" to guide the judge.

Itinerary Prompts test the trip planning wizard. These range from vague ("plan me a trip") to highly detailed ("4-day trip for a couple in December who love seafood"). The test checks that the wizard correctly identifies what information is provided vs. missing and asks the right follow-up questions. The browser agent can answer those follow-ups to test the full flow.

Guardrail Prompts test safety and brand protection. These include political questions, harmful requests, jailbreak attempts, and off-topic queries. The test verifies the Concierge deflects gracefully, doesn't leak restricted content, and redirects to travel-appropriate topics.

How Scans Work

A "scan" is a complete test run. The /test skill in Cowork launches a browser agent that visits the live Concierge, enters each prompt, waits for a response, captures it, and then uses an LLM judge to score the result. Each scan creates a test_run record in Supabase with all individual results linked to it. Scans show up in the dropdown at the top of the page.

Competitor Comparison

The competitor tab sends the same 5 itinerary prompts to Horizon Concierge, Mindtrip (mindtrip.ai), and GuideGeek (guidegeek.com). The browser agent visits each platform, enters the prompt, captures the response, and judges quality. Results are displayed side by side for direct comparison.

Scoring

Each response receives a score from 0 to 1 (displayed as 0-100%). Scores above 80% are green, 60-80% are orange, and below 60% are red. The "pass" threshold is generally 0.50 (50%). The overall score for a scan is the average across all test results.

Adding Prompts

Each test type tab (Q&A, Itineraries, Guardrails) has an "Add a prompt" form at the bottom. Enter your query and optionally an expected response. The expected response helps the LLM judge understand what a good answer looks like. Prompts persist in Supabase and will be included in future scans.

Recommendations

The Overview page includes an accordion with AI-generated recommendations based on low-scoring or failing items. These suggest concrete product improvements like integrating specific APIs, enriching the knowledge base, or adjusting wizard behavior.

Data Storage

All data is stored in Supabase (Postgres). Tables: test_runs, test_results, test_cases, test_suites, competitor_responses. The dashboard reads from these tables via the Supabase REST API using the anon key.
